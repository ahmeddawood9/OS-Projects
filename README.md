# Operating Systems Projects

This repository is a personal collection of projects built to develop a
deep, practical understanding of Operating Systems concepts through
hands-on implementation in C.

The goal of these projects is not just to make programs that work, but to
understand how real systems behave under the hood — how user programs
interact with the kernel, how files are accessed, how data flows through
buffers, and how Unix-style tools are designed.

These projects are inspired by classical Operating Systems literature
(especially OSTEP) and focus on learning by building small, focused
utilities that reflect real-world system behavior.

## Why this repository exists

- To strengthen core OS fundamentals beyond theory
- To understand Unix philosophy through implementation
- To gain confidence with low-level programming in C
- To explore how user-space programs interact with kernel abstractions
- To build a solid foundation for systems, backend, and infrastructure roles

Each project is kept small and focused, emphasizing correctness, clarity,
and alignment with Unix design principles rather than feature bloat.

More projects will be added incrementally as the learning progresses.
